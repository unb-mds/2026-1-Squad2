# 📚 Estudo Completo: Metodologias Ágeis

---

## 📖 1. O que são Metodologias Ágeis?

Metodologias ágeis são formas de desenvolver software de maneira:

- Mais rápida ⚡
- Mais flexível 🔄
- Mais colaborativa 🤝

Elas surgiram com o **Manifesto Ágil**, criado em 2001.

---

## 🧠 2. Os 4 Valores do Manifesto Ágil

1. Pessoas e interações > processos e ferramentas
2. Software funcionando > documentação extensa
3. Colaboração com o cliente > contratos rígidos
4. Responder a mudanças > seguir um plano

> 👉 **Resumo:** foco em entregar valor rápido e adaptar-se sempre.

---

## 📜 3. Os 12 Princípios Ágeis (resumo simples)

- Entregar software frequentemente
- Aceitar mudanças, mesmo no final
- Trabalhar junto com o cliente
- Manter ritmo constante
- Focar na simplicidade
- Melhorar continuamente

---

## ⚙️ 4. Como Funciona na Prática

O desenvolvimento é dividido em pequenas partes chamadas:

- **Sprint** → período curto (1 a 4 semanas)
- **Backlog** → lista de tarefas
- **Incremento** → parte do sistema pronta

---

## 🚀 5. Principais Metodologias Ágeis

### 🥇 Scrum
A mais usada no mercado.

**Como funciona:**
- Trabalho em sprints
- Reuniões:
  - **Daily** (todo dia)
  - **Planning** (planejamento)
  - **Review** (revisão)
  - **Retrospective** (melhoria)

**Papéis:**
- **Product Owner** → define o que fazer
- **Scrum Master** → organiza o processo
- **Time** → desenvolve

---

### 📊 Kanban

**Como funciona:**
- Quadro visual com colunas:
  - A fazer
  - Fazendo
  - Concluído

**Vantagens:**
- Simples
- Visual
- Fácil de usar

---

### 💻 Extreme Programming (XP)
Foco na qualidade do código.

**Práticas:**
- Testes automáticos
- Programação em dupla
- Código simples

---

## 📊 6. Vantagens das Metodologias Ágeis

- ✔ Entregas mais rápidas
- ✔ Melhor adaptação a mudanças
- ✔ Comunicação melhor
- ✔ Mais qualidade no produto

---

## ⚠️ 7. Desvantagens

- ❌ Pode ter menos documentação
- ❌ Precisa de equipe organizada
- ❌ Difícil em equipes muito grandes

---

## 🔄 8. Ágil vs Tradicional

| Tradicional | Ágil |
|---|---|
| Planeja tudo antes | Planeja aos poucos |
| Entrega no final | Entrega contínua |
| Mudanças difíceis | Mudanças fáceis |

## 🧪 10. Exemplo Real (simples)

**Projeto:** Sistema de tarefas

- **Sprint 1:**
  - Login
  - Cadastro
- **Sprint 2:**
  - Criar tarefas
  - Listar tarefas
- **Sprint 3:**
  - Editar tarefas
  - Melhorias

> 👉 A cada sprint, o sistema já funciona um pouco mais.

---

## 🎯 11. Quando Usar Metodologias Ágeis

**Use quando:**
- O projeto pode mudar
- O cliente quer participar
- Precisa de entregas rápidas

**Evite quando:**
- Os requisitos são fixos
- O projeto é muito rígido (ex: sistemas críticos)
