# Estudo de Figma + Integração com GitHub

## 1. Introdução

O **Figma** é uma ferramenta de design de interfaces amplamente utilizada para criação de protótipos interativos e colaboração em tempo real. No contexto de **Métodos de Desenvolvimento de Software (MDS)**, seu uso vai além do aspecto visual, sendo aplicado como suporte à definição, validação e comunicação de requisitos.

Além disso, o Figma pode ser integrado ao fluxo de desenvolvimento do projeto, contribuindo para maior alinhamento entre documentação, implementação e experiência do usuário.

## 2. Conceitos Fundamentais

### 2.1 UI (User Interface)

Refere-se à interface visual do sistema, incluindo elementos como botões, cores, tipografia e organização do layout. A UI é responsável pela apresentação das funcionalidades ao usuário.

### 2.2 UX (User Experience)

Relaciona-se à experiência do usuário durante a interação com o sistema. Envolve fatores como usabilidade, clareza, eficiência e facilidade de navegação.

### 2.3 Prototipação

A prototipação consiste na criação de representações visuais do sistema antes da implementação. Essas representações podem variar de **wireframes** (estrutura inicial) até interfaces de **alta fidelidade**.

Essa etapa permite validar ideias, identificar problemas e reduzir ambiguidades nos requisitos.

### 2.4 Design System

O **Design System** é um conjunto de padrões reutilizáveis, incluindo cores, tipografia e componentes. Ele garante consistência visual e facilita a manutenção e evolução do sistema.

## 3. Uso do Figma no Projeto

No contexto do projeto de MDS, o Figma é utilizado como ferramenta de apoio ao desenvolvimento, permitindo:

- Representar visualmente funcionalidades do sistema
- Criar wireframes para estruturar telas
- Desenvolver interfaces em alta fidelidade
- Construir protótipos navegáveis
- Apoiar a validação de requisitos

O foco não está apenas no design, mas na **clareza e comunicação das funcionalidades** entre os membros da equipe.

## 4. Prototipação no Contexto de MDS

A prototipação permite simular o funcionamento do sistema antes da implementação, conectando telas e definindo fluxos de interação.

No contexto da disciplina, isso possibilita:

- Visualizar o comportamento esperado de uma funcionalidade
- Validar fluxos de usuário antes do desenvolvimento
- Reduzir ambiguidades na interpretação dos requisitos

### Exemplo

**Funcionalidade:** Tela de login

- **Campos:** e-mail e senha
- **Ação:** autenticação do usuário
- **Estados:** sucesso (acesso ao sistema) e erro (mensagem de validação)

**User Story:**

> *"Como usuário, quero realizar login para acessar o sistema."*

**Critérios de aceitação:**

- Deve permitir inserção de e-mail e senha
- Deve validar campos obrigatórios
- Deve exibir mensagem de erro em caso de falha
- Deve redirecionar em caso de sucesso

## 5. Integração com GitHub e Template da Disciplina

### 5.1 Figma como apoio à documentação

O Figma complementa a documentação textual, tornando os requisitos mais visuais e de fácil compreensão.

Além disso, ele se integra ao template semanal da disciplina, sendo utilizado como apoio visual para as funcionalidades descritas.

### 5.2 Fluxo de trabalho integrado

O uso do Figma no projeto segue um fluxo integrado com GitHub e o template:

1. Definição da funcionalidade no template semanal
2. Criação ou atualização do protótipo no Figma
3. Criação de um issue no GitHub
4. Inserção do link do Figma no template e no issue
5. Implementação da funcionalidade
6. Validação com base no protótipo

Esse fluxo garante maior organização e alinhamento entre planejamento e execução.

### 5.3 Rastreabilidade entre design e desenvolvimento

A utilização conjunta de Figma, template e GitHub permite estabelecer rastreabilidade entre:

- Requisitos definidos
- Representação visual (protótipo)
- Implementação no código

Isso facilita a identificação de mudanças, reduz inconsistências e melhora a comunicação da equipe.

## 6. Benefícios no Projeto MDS

A utilização do Figma integrada ao fluxo do projeto traz diversos benefícios:

- Redução de ambiguidades nos requisitos
- Melhor comunicação entre os membros da equipe
- Apoio visual para desenvolvimento das funcionalidades
- Maior consistência na interface do sistema
- Facilidade na validação antes da implementação
- Integração com o processo definido na disciplina

## 7. Conclusão

O uso do Figma no projeto de MDS contribui para um desenvolvimento mais estruturado, atuando como uma **ponte entre requisitos, design e implementação**.

Sua integração com o GitHub e com o template semanal da disciplina permite alinhar o planejamento com a execução, reduzindo erros e melhorando a qualidade final do sistema.

Dessa forma, o Figma não se limita ao design de interfaces, mas se torna uma ferramenta estratégica no processo de desenvolvimento de software.
